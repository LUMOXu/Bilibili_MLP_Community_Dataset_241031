# `1_video` 文件夹
这个文件夹包含了报告**第一部分：视频**的原始数据。
## `video_info.csv`
所有`157,057`个MLP视频的基本信息（数据统计于2024年11月初）

视频按照投稿时间排序。

### 收录标准：

**1.** 在`2024/10/31 23:59:59`前发布

**2.** B站搜索`小马宝莉` `彩虹小马` `我的小马驹` `mlp` `My Little Pony`五个关键词能搜索到的视频。由于B站搜索功能的限制，一些没有加相关tag的视频无法被统计到。

### 各列含义：
- **`idx`**  
  数据行编号。

- **`bvid`**  
  BV号。

- **`aid`**  
  AV号。

- **`videos`**  
  分p数量。

- **`tid`**  
  视频所属分区的数值 ID。

- **`tname`**  
  视频所属分区名称（如“连载动画”、“鬼畜”等）。

- **`copyright`**  
  视频版权类型：`1` 表示自制，`2` 表示转载。

- **`pic`**  
  视频封面图像链接。

- **`title`**  
  视频标题。

- **`pubdate`**  
  视频发布时间（UNIX时间戳）。

- **`fmt_pubdate`**  
  视频发布时间（格式化日期字符串，如`2011/2/27 2:39:02`）。

- **`ctime`**  
  视频上传时间（UNIX时间戳）。基本只有定时发布的视频，两个时间才不同。

  注意，早期的视频没有这个数据。

- **`fmt_ctime`**  
  视频上传时间（格式化日期字符串）。

- **`desc`**  
  视频简介。

- **`duration`**  
  视频时长，单位为秒。

- **`dynamic`**  
  配套动态内容。

- **`owner_mid`**  
  视频投稿者的 UID。

- **`owner_name`**  
  视频投稿者的昵称。

- **`owner_face`**  
  投稿者头像链接。

- **`view`**  
  视频播放次数。

- **`danmaku`**  
  弹幕数量。

- **`reply`**  
  评论数量。

- **`favorite`**  
  收藏次数。

- **`coin`**  
  投币次数。

- **`share`**  
  分享次数。

- **`his_rank`**  
  如果上过全站排行榜，上榜最高排名，`0` 表示未上榜。

- **`like`**  
  点赞数量。

## `raw_video_data.7z`
视频的原始数据。为`bilibili_api`包`video.Video.get_info()`方法返回的原始json

每1000个视频的数据合并成一个大json文件，里面有视频原始信息。

包含了第一批14万个视频（`小马宝莉`关键词）和之后1万个视频（其余关键词新搜到的）的两批原始信息。

注意，由于数据分成两批，编号与`video_info.csv`中的`idx`列不对应。

数据较原始，复杂，请自行探索使用方法。可以从这个原始数据构建`video_info.csv`。

## `tags.7z`
视频tag的原始数据。为`bilibili_api`包`video.Video.get_tags()`方法返回的原始json

每1000个视频的数据合并成一个大json文件，里面有tag信息。

这个文件中编号与`video_info.csv`中的`idx`列可以对应。

# `2_up` 文件夹

这个文件夹包含了报告**第二部分：UP主**的原始数据。

## `up_info.csv`

`5,507`位MLP up的基本信息。（数据统计于2024年11月初）

up主按照粉丝数降序排序。
### 收录标准：
在`2024/10/31 23:59:59`前...

**1.** 投稿过至少三个MLP视频；

**2.** 粉丝数大于10；

**3.** 投稿的MLP视频占所有视频的至少4%。

### 各列含义：
- **`uid`**  
  用户的唯一 ID 编号。

- **`name`**  
  UP 主的昵称。

- **`cluster`**  
  聚类得出的类别编号，注意与文档中的编号不同，但各类的人数相同。

- **`face`**  
  头像图片链接。

- **`mlp_vids`**  
  该 UP 主发布的与MLP相关的视频数量。

- **`repost`**  
  该 UP 主转载视频数量。

- **`not_repost`**  
  自制视频数量。

- **`repost_pct`**  
  转载视频所占的比例（百分比）。

- **`follower`**  
  该 UP 主的粉丝数量。

- **`following`**  
  该 UP 主关注的其他用户数量。

- **`total_vids`**  
  发布的总视频数量。

- **`mlp_pct`**  
  与MLP相关的视频占该 UP 主总视频数的百分比。

- **`sex`**  
  UP 主性别（如男性、女性或未知）。

- **`sign`**  
  个性签名或简介。

- **`level`**  
  Bilibili 等级。

- **`silence`**  
  是否被封禁。

- **`vip_type`**  
  会员类型。

- **`vip_status`**  
  当前 VIP 状态。

- **`top_photo`**  
  主页顶部背景图链接。

- **`birthday`**  
  UP 主的生日信息。

- **`school`**  
  所属学校信息（如果有填写）。

- **`tags`**  
  该 UP 主在平台中活跃的标签关键词集合。

- **`view`**  
  所有视频的总播放量。

- **`like`**  
  所有视频的总点赞数。

- **`earliest_pubdate`**  
  该 UP 主最早发布的视频的 Unix 时间戳。

- **`latest_pubdate`**  
  最近一次视频发布时间的 Unix 时间戳。

- **`earliest_fmt_pubdate`**  
  最早视频的格式化日期时间。

- **`latest_fmt_pubdate`**  
  最近视频的格式化日期时间。

- **`mlp_view`**  
  与MLP相关的视频总播放量。

- **`mlp_like`**  
  与MLP相关的视频总点赞数。

- **`mlp_coin`**  
  与MLP相关的视频总投币数。

- **`mlp_view_pct`**  
  MLP视频播放量占总播放量的百分比。

- **`mlp_like_pct`**  
  MLP视频点赞数占总点赞数的百分比。

- **`following_mlp`**  
  没有实装

- **`follower_mlp`**  
  没有实装

- **`active`**  
  活跃度指标。为0则表示近半年没有发过MLP视频，否则为上次发MLP视频过去的天数。

## `raw_up_info.csv`
这个csv展示了所有投稿过MLP视频的`32,615`名up。只有`uid` `name` `face` 	`mlp_vids` `repost` `not_repost` `repost_pct`七列。

## `raw_up_data.csv`
up主信息的原始数据。
内有四个文件夹：
- **`get_info`**  
  获取up的详细个人信息（当然，是公开的）。

- **`get_relation_info`**  
 获取up的关注人数与粉丝数。

- **`get_up_stat`**  
  获取up所有视频的总播放与总点赞数。

- **`get_video`**  
  这个函数理论上可以多次调用，获得up的所有视频，但我们只调用了一次，返回了第一页的视频内容。目的只是为了获取up投稿的总视频数。
  
四个文件夹的内容均为`bilibili_api`包`user.User.对应函数名()`方法返回的原始json。

每100个up的数据合并成一个大json文件，里面有原始信息。

四个文件夹里的up主编号可与`raw_up_info.csv`的行编号对应。

数据较原始，复杂，请自行探索使用方法。可以从这些原始数据构建`up_info.csv`。

## `mlpcmt_usr_2411.csv`
这个文件汇总了在B站MLP相关视频下评论用户的行为数据。总共统计了`368,371`名评论过MLP视频的用户。

**注意：** 统计范围是`5,507`位MLP up投稿的MLP视频（`112,509`个），而不是**所有**up投稿的MLP视频（`157,057`个）。

每一行代表一位用户的评论活动统计信息，包括评论数量、被UP主点赞/回复的情况、总点赞与回复数、以及首次与最近评论的时间。

### 各列含义：

- **`uid`**  
  用户的唯一 ID 编号。

- **`name`**  
  用户昵称。

- **`comment`**  
  用户发布的评论总数。

- **`root`**  
  用户发布的根评论数量（即非回复，直接评论视频的数量）。

- **`child`**  
  用户发布的子评论数量（回复他人评论的数量）。
  
  **注意，我们在爬取时，一条根评论最多爬取3条子评论，所以这个数严重低估。**

- **`up_like`**  
  用户评论收到的 UP 主点赞次数。

- **`up_reply`**  
  用户评论收到的 UP 主回复数量。

- **`like`**  
  用户评论收到的总点赞数。

- **`reply`**  
  用户评论收到的总回复数 **（同理，严重低估）**。

- **`fmt_first_cmt`**  
  用户第一条评论的格式化时间。

- **`fmt_last_cmt`**  
  用户最近一条评论的格式化时间。

- **`first_cmt`**  
  用户第一条评论的 Unix 时间戳。

- **`last_cmt`**  
  用户最近一条评论的 Unix 时间戳。
